## Chat Completion

Stateless chat completion — send a full `messages` array and get the AI response directly. No polling required.

```
POST /api/v1/chat
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `messages` | array | Yes | Array of message objects with `role` and `content` |
| `model` | string | No | Model identifier (default: `"auto"` — selects best model) |

Each message object:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `role` | string | Yes | One of `"system"`, `"user"`, or `"assistant"` |
| `content` | string | Yes | The message content |

The last message in the array must have `role: "user"`.

### Examples

#### Basic

```bash
curl -X POST https://api.kolbo.ai/api/v1/chat \
  -H "X-API-Key: kolbo_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "Explain quantum computing in simple terms"}
    ]
  }'
```

#### With System Prompt and Model

```bash
curl -X POST https://api.kolbo.ai/api/v1/chat \
  -H "X-API-Key: kolbo_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "system", "content": "You are a senior Python developer. Write clean, well-documented code."},
      {"role": "user", "content": "Write a function to sort a list using quicksort"}
    ],
    "model": "gpt-4o"
  }'
```

#### Multi-Turn Conversation

Since the API is stateless, include the full conversation history in each request:

```bash
curl -X POST https://api.kolbo.ai/api/v1/chat \
  -H "X-API-Key: kolbo_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "system", "content": "You are a helpful assistant"},
      {"role": "user", "content": "What is machine learning?"},
      {"role": "assistant", "content": "Machine learning is a subset of AI..."},
      {"role": "user", "content": "Give me a practical example"}
    ]
  }'
```

#### JavaScript

```javascript
const response = await fetch('https://api.kolbo.ai/api/v1/chat', {
  method: 'POST',
  headers: {
    'X-API-Key': process.env.KOLBO_API_KEY,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    messages: [
      { role: 'system', content: 'You are a helpful coding assistant' },
      { role: 'user', content: 'Explain the theory of relativity' }
    ],
    model: 'gpt-4o'
  })
});

const data = await response.json();
console.log(data.content);
// "The theory of relativity, developed by Albert Einstein..."
console.log(`Credits used: ${data.credits_used}`);
```

#### Python

```python
import requests

r = requests.post("https://api.kolbo.ai/api/v1/chat",
    headers={"X-API-Key": "kolbo_live_..."},
    json={
        "messages": [
            {"role": "user", "content": "Explain the theory of relativity"}
        ],
        "model": "gpt-4o"
    })
data = r.json()
print(data["content"])
print(f"Credits used: {data['credits_used']}")
```

  "success": true,

## Session-Based Chat (Advanced)

For server-managed conversation sessions, use the conversations endpoint. Messages are stored server-side and can be listed later.

```
POST /api/v1/chat/conversations
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | string | Yes | The message to send to the AI model |
| `model` | string | No | Model identifier (default: auto-select best model) |
| `session_id` | string | No | Continue an existing conversation. Omit to start a new one |
| `system_prompt` | string | No | System prompt for new conversations (ignored if `session_id` is provided) |
| `enhance_prompt` | boolean | No | Enhance prompt for better results (default: `true`) |
| `web_search` | boolean | No | Enable web search for up-to-date information (default: `false`) |
| `deep_think` | boolean | No | Enable deep thinking/reasoning mode (default: `false`) |

### Response

```json
{
  "success": true,
  "message_id": "abc123",
  "session_id": "def456",
  "type": "chat",
  "model": "auto",
  "poll_url": "/api/v1/generate/abc123/status",
  "poll_interval_hint": 2
}
```

Poll the `poll_url` to get the AI response when ready.

## List Conversations

```
GET /api/v1/chat/conversations
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `page` | number | Page number (default: 1) |
| `limit` | number | Results per page (default: 20, max: 50) |

```bash
curl "https://api.kolbo.ai/api/v1/chat/conversations" \
  -H "X-API-Key: kolbo_live_..."
```

## Finding Models

Use the [Models endpoint](/docs/developer-api/models-and-pricing) to discover available chat models:

```bash
curl "https://api.kolbo.ai/api/v1/models?type=chat" \
  -H "X-API-Key: kolbo_live_..."
```

## Tips

- **Auto-select**: Omit the `model` field to let Kolbo choose the best model automatically.
- **Stateless vs Session-based**: Use `POST /api/v1/chat` for simple request/response. Use `POST /api/v1/chat/conversations` if you need server-side conversation history.
- **Context window**: The stateless endpoint sends the full messages array to the AI. Keep conversations concise for best results and lower costs.
